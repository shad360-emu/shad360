# Security Policy

## Supported Versions

Security fixes are provided for the latest release and the current development branch (`main`).

| Version | Supported |
| -------- | :-------: |
| Latest Release | ✅ |
| `main` | ✅ |
| Older Releases | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in shad360, please report it **privately**.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, use one of the following methods:

- GitHub Security Advisories (preferred)
- Email the maintainers (if available)
- Contact a maintainer privately

When reporting a vulnerability, please include:

- A description of the issue
- Steps to reproduce
- A proof of concept (if applicable)
- The affected version or commit
- Your operating system and architecture
- Any relevant logs, screenshots, or sample files

## Response Process

Our goal is to:

- Acknowledge reports within **7 days**.
- Investigate and verify the issue.
- Develop and test a fix.
- Release a security update as soon as practical.
- Credit the reporter in the release notes (unless they prefer to remain anonymous).

Response times may vary depending on the complexity of the issue.

## Responsible Disclosure

Please allow us reasonable time to investigate and fix the issue before publicly disclosing it.

Responsible disclosure helps protect users while a fix is being prepared.

## Scope

Examples of security vulnerabilities include:

- Remote code execution
- Privilege escalation
- Memory corruption that could be exploited
- Arbitrary file read or write
- Command injection
- Dependency-related vulnerabilities

The following are generally **not** considered security issues:

- Emulator crashes
- Game compatibility problems
- Performance issues
- Rendering or audio bugs
- Incorrect emulation
- Build failures
- Feature requests

If you are unsure whether something qualifies as a security issue, feel free to report it privately.

## Third-Party Components

shad360 includes third-party software.

If a reported vulnerability exists in an upstream dependency, we may coordinate with the upstream project before releasing a fix.

## Security Updates

Security fixes are included in the next available release. Critical issues may receive an out-of-band release if necessary.

## Acknowledgements

Thank you to everyone who helps keep shad360 secure by reporting vulnerabilities responsibly.
